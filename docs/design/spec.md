```typescript
  dailyAdvice: DailyAdvice | null;

}

  date: string;
  lunarDate: string;
  overallFortune: string;
  workFortune: {
    mCreate: string;      // エムクリエイト向け
```

#### 動作フロー

1. ページ読み込み時、IndexedDBから当日のキャッシュを検索
2. キャッシュがあればそのまま表示（API呼び出しなし）
3. キャッシュがなければ `/api/daily-advice` を呼び出し
4. 生成結果をIndexedDBにキャッシュ保存
5. 日付が変わったらキャッシュを無効化

|--------------|------|
| `ChatHeader` | タイトル + 新規会話・履歴ボタン |
| `ChatMessageList` | メッセージ一覧（スクロール可能） |

#### 状態管理

```typescript
interface ChatState {
  conversationId: string;
  messages: ChatMessage[];
  isStreaming: boolean;
  selectedCategory: 'mcreate' | 'jybee' | 'private' | 'general';
  error: string | null;
}

interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  timestamp: string;
  category: string;
  references?: KnowledgeReference[];
}

interface KnowledgeReference {
  source: string;        // 例: "kantei-transcript-full.txt"
  label: string;         // 例: "プロ鑑定(2025/12)"
  relevanceScore: number;
  excerpt: string;       // 該当箇所の抜粋
}
```

#### カテゴリ選択の効果

| カテゴリ | システムプロンプト追加指示 | RAG検索の重み付け |
|---------|-------------------------|-----------------|
| エムクリエイト | 事業1（SNSコンサル）の文脈で回答 | 事業系ナレッジ優先 |
| JYビー協会 | 事業2（陰陽五行美容）の文脈で回答 | 美容・五行系ナレッジ優先 |
| プライベート | 個人生活の文脈で回答 | パートナーシップ・旅行系優先 |
| 一般 | カテゴリ指定なし | 全ナレッジ均等 |

#### ストリーミング動作

1. ユーザーメッセージ送信
2. 送信ボタンを無効化、AIバブルに「...」表示
3. `/api/chat` へ POST（ストリーミング）
4. Server-Sent Events で逐次テキスト受信
5. AIバブルにリアルタイム表示
6. ストリーミング完了後、参照タグを表示
7. 送信ボタンを有効化
8. 会話をIndexedDBに自動保存

### 3.2 POST `/api/chat`

AI相談チャットエンドポイント。Server-Sent Events によるストリーミング応答。

#### リクエスト

```typescript
interface ChatRequest {
  conversationId: string;
  message: string;
  category: 'mcreate' | 'jybee' | 'private' | 'general';
  history: {
    role: 'user' | 'assistant';
    content: string;
  }[];
}
```

#### レスポンス（SSE）

```
Content-Type: text/event-stream

data: {"type": "text", "content": "円香さんの"}
data: {"type": "text", "content": "命盤を拝見すると"}
...
data: {"type": "references", "references": [...]}
data: {"type": "done"}
```

#### 処理フロー

```
1. リクエスト受信
   │
2. カテゴリに基づきRAG検索クエリを構築
   │
3. /api/knowledge/search でナレッジ検索
   │  ├── 質問テキストを埋め込みベクトル化
   │  ├── ベクトルDBで類似検索（上位5件）
   │  └── 関連ナレッジチャンクを取得
   │
4. システムプロンプトを構築
   │  ├── 基本プロンプト（常時）
   │  ├── 命盤コアデータ（常時）
   │  ├── カテゴリ別追加指示
   │  └── RAG検索結果をコンテキストとして注入
   │
5. Claude API 呼び出し（ストリーミング）
   │  ├── model: "claude-sonnet-4-6-20250514"
   │  ├── max_tokens: 2048
   │  └── stream: true
   │
6. SSEでクライアントにストリーミング送信
   │
7. 完了時に参照情報を送信
```

### 3.3 GET `/api/daily-advice`

#### レスポンス

```typescript

  lunarDate: string;
```

#### 処理フロー

```

   │

   │  ├── 西暦日付 → 農暦変換
   │  ├── 日干支の算出
   │  └── 流日星の命盤配置算出
   │
3. RAG検索（当日の運勢に関連するナレッジ）
   │
4. プロンプト構築
   │  ├── 命盤コアデータ
   │  ├── 当日の干支・流日星情報
   │  ├── 現在の大限・年運情報
   │  ├── RAG検索結果
   │  └── 出力フォーマット指定（JSON）
   │
5. Claude API 呼び出し（非ストリーミング）
   │
6. JSON形式でレスポンス返却
```

## 4. データモデル（IndexedDB スキーマ）

### 4.1 Dexie.js データベース定義

```typescript
import Dexie, { type Table } from 'dexie';

class FortuneTellingDB extends Dexie {
  conversations!: Table<Conversation>;
  messages!: Table<Message>;
  dailyAdviceCache!: Table<DailyAdviceCache>;
  bookmarks!: Table<Bookmark>;
  settings!: Table<Setting>;

  constructor() {
    super('FortuneTellingDB');
    this.version(1).stores({
      conversations: '++id, createdAt, category, title',
      messages: '++id, conversationId, role, timestamp, [conversationId+timestamp]',
      dailyAdviceCache: 'date, generatedAt',
      bookmarks: '++id, messageId, conversationId, createdAt',
      settings: 'key',
    });
  }
}
```

### 4.2 テーブル定義

#### conversations

| カラム | 型 | 説明 | インデックス |
|--------|-----|------|------------|
| id | number (auto) | 会話ID | PK |
| title | string | 会話タイトル（最初の質問から自動生成） | ○ |
| category | string | カテゴリ | ○ |
| createdAt | string (ISO) | 作成日時 | ○ |
| updatedAt | string (ISO) | 更新日時 | - |
| messageCount | number | メッセージ数 | - |

#### messages

| カラム | 型 | 説明 | インデックス |
|--------|-----|------|------------|
| id | number (auto) | メッセージID | PK |
| conversationId | number | 会話ID | ○ |
| role | 'user' \| 'assistant' | 発言者 | - |
| content | string | メッセージ内容 | - |
| timestamp | string (ISO) | タイムスタンプ | ○ |
| references | KnowledgeReference[] | ナレッジ参照（assistant のみ） | - |

#### dailyAdviceCache

| カラム | 型 | 説明 | インデックス |
|--------|-----|------|------------|
| date | string (YYYY-MM-DD) | 日付 | PK |
| advice | DailyAdvice | アドバイスデータ | - |
| generatedAt | string (ISO) | 生成日時 | ○ |

#### bookmarks

| カラム | 型 | 説明 | インデックス |
|--------|-----|------|------------|
| id | number (auto) | ブックマークID | PK |
| messageId | number | メッセージID | ○ |
| conversationId | number | 会話ID | ○ |
| note | string | メモ | - |
| createdAt | string (ISO) | 作成日時 | ○ |

#### settings

| カラム | 型 | 説明 | インデックス |
|--------|-----|------|------------|
| key | string | 設定キー | PK |
| value | any | 設定値 | - |

## 5. AIプロンプト設計

### 5.1 システムプロンプト構造

システムプロンプトは以下の4層で構成する。

```
┌─────────────────────────────────────────────┐
│ Layer 1: ロール定義 & ガイドライン             │
│ （固定、常時ロード）                           │
├─────────────────────────────────────────────┤
│ Layer 2: 命盤コアデータ                        │
│ （固定、meiban-core.md から自動ロード）         │
├─────────────────────────────────────────────┤
│ Layer 3: カテゴリ別追加指示                     │
│ （動的、ユーザー選択カテゴリに応じて切り替え）    │
├─────────────────────────────────────────────┤
│ Layer 4: RAGコンテキスト                       │
│ （動的、質問に関連するナレッジ検索結果）         │
└─────────────────────────────────────────────┘
```

### 5.2 Layer 1: ロール定義 & ガイドライン

```
あなたは紫微斗数を中心とした東洋占術の専門AIアドバイザーです。
円香（まどか）さん専属のアドバイザーとして、命盤データとプロの鑑定士の
アドバイスに基づいて的確な助言を行います。

## 回答ガイドライン

1. 必ず円香さんの命盤データに基づいて回答してください
2. プロの鑑定士のアドバイスがある場合は優先的に参照し、出典を明示してください
3. 紫微斗数の専門用語を使う場合は、簡潔な説明を添えてください
4. 事業面の相談には具体的な行動提案を含めてください
5. プライベートの相談には命盤の性質を踏まえた共感的な回答をしてください
6. 「あなたの性質的に〜」という形式で、命盤の性質を活かしたアドバイスを心がけてください
7. 不確かな情報は推測であることを明示してください
8. 鑑定データにない内容は「この点についてはプロの鑑定士に確認されることをお勧めします」と案内してください

## 回答トーン

- 親しみやすく、敬語を使う
- 占術の専門家として自信を持って回答
- 相談者に寄り添う姿勢
- 具体的で実践的なアドバイス
```

### 5.3 Layer 2: 命盤コアデータ

`data/knowledge/meiban-core.md` の全文をそのままシステムプロンプトに含める。以下のセクションが常時参照可能になる:

## 7. ナレッジ管理フロー

### 7.1 ナレッジの種別

| 種別 | 格納先 | ロード方式 | 更新頻度 |
|------|--------|-----------|---------|
| コアナレッジ | `data/knowledge/meiban-core.md` | システムプロンプトに全文ロード | 不変 |
| RAGナレッジ | `data/knowledge/*.md, *.txt` | ベクトル検索で関連部分のみロード | 鑑定ごと |
| 年間データ | `data/yearly/*.json` | 構造化データとして直接参照 | 年1回 |

### 7.2 ナレッジ追加手順（開発者向け）

```
1. 新しい鑑定データ（文字起こし or 書類）を入手

2. テキスト化
   └── PDF → Markdown に変換
   └── 音声文字起こし → テキストファイルに整形

3. ファイルを配置
   └── data/knowledge/{filename}.md

4. インデクシング実行
   └── pnpm run index-knowledge
       ├── テキストをチャンクに分割（512トークン、128オーバーラップ）
       ├── 各チャンクを埋め込みベクトルに変換
       ├── メタデータ付与（ファイル名、カテゴリ、日付）
       └── ベクトルDB（data/vector-store/）に保存

5. 動作確認
   └── チャットで関連質問をして参照されるか確認
```

### 7.3 ナレッジ追加手順（アプリUI経由）

```
1. 設定画面 → 「ナレッジを追加」ボタン

2. ファイル選択ダイアログ
   └── 対応形式: .md, .txt

3. カテゴリ選択
   └── 鑑定データ / 方位データ / その他

4. アップロード実行
   └── POST /api/knowledge/upload
       ├── ファイルをサーバーに保存
       ├── チャンク分割 + ベクトル化
       └── ベクトルDBに登録

5. 一覧に追加表示
```

- 目的地入力 → 方位自動判定

- カテゴリフィルター
- ブックマーク一覧表示
